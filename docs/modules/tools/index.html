Tools

Schematix Initializer

To make it easier to get started with a Schematix Schema, you can run the Schematix Initializer, which will create a basic schema for you.

You can run the initializer by running:

./schematix init

After that, save the resulting schema in a file of your choice, for example initializer.schema.json, and this should give a good starting point to further customize the schema.

If you want to initialize a new project with a list of properties, it's also possible to provide the --source argument, which takes in a newline separated file consisting of a list of property names.

For example, give a file named properties.csv, which contains the following contents:

firstname
lastname
income

The command

./schematix init --source=properties.csv

Will generate a new schema with the above properties in the generated schema. Note that at this moment it's only possible to generate a schema with a list of optional strings.

Generate Schema from JSON file

Schematix provides an easy way to generate a schema without having to write it by hand.

Based on the contents a file called my-data.json (see below):

{
  "transaction_date": "123",
  "reference": "tUEJRjXA4fAYmvfR8tGcYkQ",
  "amount": 52.70340131765968,
  "account": "910623899"
}

When invoking the following command:

./schematix generate-schema my-data.json

Schematix will generate this schema:

{
  "description": "Generated at <date> based on transaction.json",
  "title": "transaction",
  "type": "object",
  "properties": {
    "transaction_date": {
      "type": "string"
    },
    "reference": {
      "type": "string"
    },
    "amount": {
      "type": "number"
    },
    "account": {
      "type": "string"
    }
  }
}

Generate Schema from CSV file

Similar to generating a schema from a JSON file, you can generate a schema from a CSV file.

Based on the contents a file called my-data.csv (see below):

transaction_date,reference,amount,account
12,tUEJRjXA4fAYmvfR8tGcYkQ, 52.70340131765968,910623899

When invoking the following command:

./schematix generate-schema my-data.csv

Schematix will generate this schema:

{
  "description": "Generated at <date> based on transaction.csv",
  "title": "transaction",
  "type": "object",
  "properties": {
    "transaction_date": {
      "type": "integer"
    },
    "reference": {
      "type": "string"
    },
    "amount": {
      "type": "number"
    },
    "account": {
      "type": "string"
    }
  }
}

Generate SQL from JSON Schema

Schematix provides tooling to turn a JSON schema into a SQL DDL file without having to write it by hand. Depending on the complexity of the schema, this can save quite a bit of time.

To invoke the SQL generation, you can invoke the following command:

./schematix generate-sql my.schema.json --dialect=SQLServer --output=schema.sql

Executing the above will generate a schema.sql file with both drop and create statements for the schema. An example can be seen below:

drop table entity_demo;

create table entity_demo (
   id int identity not null,
    name varchar(255),
    transaction_date varchar(255),
    amount float,
    cards int,
    active bit,
    cardtype varchar(255),
    primary key (id)
);

Schematix SQL generation supports many SQL dialects, such as:

  • Oracle
  • MySQL
  • SQL Server
  • H2

For a full list of supported dialects, pleas check the Hibernate Dialects Documentation.

Inspect fixed width lines

Handling fixed width files is often a challenge in the way how they are formatted, and as such, it's often hard to see which field is which.

For example, when dealing with a text such as 202001101001, it's hard to find the significance of the fields embedded in the input string. However, using Schematix Inpect, dealing with input like this becomes much easier.

When involving the following command:

./schematix inspect --mapping=my.mapping.json 202001101001

The following output will be generated by Inspect:

date    = 20200110 
version = 10
status  = 01
