Documentation

Documentation Generation#

Zero effort documentation creation.

Overview#

At Schematix we understand that while schemas are of great importance when exchanging data, the format in which the schemas are written in are not for everyone.

To address this, you can use the Schematix documentation generator, which transforms technical schemas effortlessly into human readable documentation, including Word, PDF and plain text support (Asciidoctor).

Basic usage#

A simple way to generate the documentation is as follows. Given the following schema (claims.schema.json):

{
"$schema": "http://json-schema.org/draft-07/schema#",
"$id": "http://schematix.net/medical_claims.schema.json",
"title": "Medical Claims (5.0)",
"description": "Version 5.0 of the medical claim structure",
"type": "object",
"properties": {
"provider number": {
"description": "A unique provider number",
"type": "string"
},
"membership number": {
"description": "A unique memebership number",
"type": "string",
"examples": [
"1203",
"09013",
"3892"
]
},
"patient id": {
"description": "A 12 digit code describing the unique patient id",
"type": "string",
"examples": [
"JKW123094732"
]
}
}
}
./schematix generate-documentation claims.schema.json

Schematix will read the Claims Schema and turn the information into human readable documentation.

Items suchs as:

  • Field names
  • Data types
  • Fields being mandatory or not
  • Field descriptions, including example values
