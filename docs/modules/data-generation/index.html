Data Generation

Data Generation

Work with limitless Personally Identifiable Information (PII) in the most secure way.

Overview

In the world of Data exchange, having the data to test, experiment, explorer is a must have. But dealing with data brings changellenges, such as the availability, correctness and privacy aspects aspects of data.

Schematix Data Generation addresses those issues by providing the capability to generate real looking data according to well defined rules. This prevents leaking privacy related data, or having to spend large quantities of time on the collection and filtering of data.

Within seconds, Schematix can generate the right amount of data in exactly the format you want.

Basic usage

./schematix generate-data example.schema.json

Generation Strategies

Schematix has several data generation strategies. When generating data for properties defined in the schema, the sequence of generation strategies determines which strategy has preference over the next.

The default order of strategies defined are the following:

  1. Override Strategy
  2. Natural Strategy
  3. Schema Strategy

This means, that when generating data for a property defined in the schema, Schematix will first look at the Override Strategy, followed by the Natural Strategy, and then the Schema Strategy.

Besides this, there are several optional strategies, which won't be applied unless specifically defined.

Natural

The Natural Strategy is used to generate real looking data, such as phone numbers, names, credit cards, and much more.

The Natural Strategy determines the strategy to use based on the property name, and will try to find a generator based on that name. For example, when a property has been defined with the name firstName, a random first name will be generated.

Input

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "https://schema.schematix.net/documentation/simple-types.json",
  "type": "object",
  "properties": {
    "firstName": {
      "type": "string"
    },
    "lastName": {
      "type": "string"
    },
    "phoneNumber": {
      "type": "string"
    },
    "birthDay": {
      "type": "string"
    }
  }
}

Schema Based

Schema looks at the schema.

Example Based

The example based strategy looks at the defined examples, and picks one from them. This can be used when you want to have more real like data, and can't use an enum for the schema (for example, because the number of possible values is too large or undefined)

To define an example, you can define a property with the following syntax:

"name": {
  "x-schematix" : {
    "generation-strategy": "example"
  },
  "type": "string",
  "examples": ["duck", "goose"]
},
note

The example strategy will only be used when it's explicitly defined in an override, either using x-schematix or by setting the generation-strategy globally.

Override Strategy

To have more control over which generation strategy Schematix uses, it's possible to override the strategy being used.

To override the strategy, you can set the custom property generation-strategy (part of the x-schematix property) in the schema, which will instruct the Data Generation module to prefer an alternative generation strategy. The possible values of generation-strategy are:

  • natural
  • schema
  • example

Advanced usage

This section covers the more advanced topic.

Decimal precision

By default, Schematix uses two decimals for floating point numbers when generating data. However, it's possible to customise this, either on a property level or on a global level (ie. for all decimal number in the schema)

To specify the number of decimal, you can adjust the custom property scale (part of the x-schematix property) in the schema, like this:

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "https://schema.schematix.net/data-gen/simple_types.json",
  "type": "object",
  "properties": {
    "amount": {
      "x-schematix" : {
        "scale": "5"
      },
      "type": "number"
    }
  }
}

This will generate values with a precision of 5, for example: 100.43903.

The scale option can also be specified globally via the command line, by providing the --scale argument, for example --scale=3.

Locale usage

When generating data, Schematix implies the data should be created using a US locale. However, it's possible to override this by providing a language using the --language parameter.

For example, when using the Dutch locale (nl), Schematix will generate Dutch names and Dutch phone numbers:

./schematix generate-data example.schema.json --language=nl

Full example

This example demonstrates most of capabilities of Schematix data generation:

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "https://schema.schematix.net/data-gen/simple_types.json",
  "type": "object",
  "properties": {
    "age": {
      "x-schematix" : {
        "generation-strategy": "natural"
      },
      "type": "string"
    },
    "name": {
      "x-schematix" : {
        "generation-strategy": "schema"
      },
      "type": "string",
      "enum": ["duck", "goose"]
    },
    "breed": {
      "type": "string",
      "x-schematix" : {
        "generation-strategy": "example",
        "scale" : 1
      },
      "pattern": "[a-z]+",
      "examples": [
        "bulldog",
        "pug",
        "poodle"
      ]
    }
  }
}

Incorrect data generation

Sometimes you may want to generate incorrect data, for example, for testing purposes. You can do that using:

./schematix generate-data example.schema.json --error-rate=5

This will generate 95% correct data and 5% incorrect data.
